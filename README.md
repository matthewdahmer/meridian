# Meridian

Chandra X-ray Observatory spacecraft thermal visualization suite. Meridian is a static multi-page website composed of two tools:

- **Pitch Sensitivity (Protractor)** — shows the maximum allowed dwell time per thermal model as a function of pitch angle, under coupled thermal constraints. Rendered as a semicircular polar plot with color-coded constraint bands.
- **Steady States** — shows Xija model steady-state temperatures as multi-line dwell curves and summary tables, one model at a time, across pitch angles and mission dates.

Both apps share the same dark sidebar theme, Bootstrap 5, and D3 v7. All vendor assets are served locally — no CDN dependencies.

## Development Server

No build step is required for development. Serve the repository root with Python's built-in HTTP server:

```bash
cd meridian/
python -m http.server 8765
```

Then open:

| App | URL |
|-----|-----|
| Pitch Sensitivity | http://localhost:8765/protractor/ |
| Steady States | http://localhost:8765/steady_states/ |

> **Hard refresh:** `Cmd+Shift+R` (macOS) after editing `app.js` to bypass the browser cache.

## Generating Steady States Data

The Steady States app requires data files generated by the `xija_steady_states` package, which lives as a sibling directory at `../xija_steady_states/`. It is not assumed to be installed — run it directly from its source tree. This step requires the `ska3-dev` conda environment and access to the Chandra engineering archive.

```bash
conda activate ska3-dev
cd ../xija_steady_states/
SKA=~/AXAFDATA/SKADATA python -m xija_steady_states --output-dir ../meridian/steady_states
```

This generates `meridian/steady_states/data/` containing:
- `manifest.json` — list of all 16 thermal models
- `<nav_id>/metadata.json` — per-model config and file index
- `<nav_id>/<nav_id>_<config_id>_<date>.json` — per-date chart data (28 dates per config)
- `<nav_id>/<nav_id>_<config_id>_<date>.csv` — per-date dwell CSV
- `<nav_id>/<nav_id>_<config_id>_table.csv` — summary table across all dates

Generation covers all 16 models and all 40 configurations (roughly 30–60 minutes). To restrict to specific models or dates:

```bash
SKA=~/AXAFDATA/SKADATA python -m xija_steady_states \
    --output-dir ../meridian/steady_states \
    --models aca dpa fptemp \
    --dates 2026001 2026182
```

To regenerate without skipping existing files:

```bash
SKA=~/AXAFDATA/SKADATA python -m xija_steady_states \
    --output-dir ../meridian/steady_states \
    --force
```

The protractor's `data/` directory is managed separately and is not generated by this command.

## Building for Deployment

`build.py` assembles both apps into a single `dist/` directory ready to copy to a web server.

```bash
cd meridian/
python build.py
```

This copies `protractor/`, `data/`, and `steady_states/` (including generated data) into `dist/`. Run with `--force` to overwrite an existing `dist/`:

```bash
python build.py --force
```

To build into a different directory:

```bash
python build.py --output-dir /path/to/webroot --force
```

Verify the result by serving it locally:

```bash
python -m http.server 8765 --directory dist
```

> `build.py` warns but does not fail if `steady_states/data/` is absent — generate the data first before a production build.

## Testing

### Steady States (JavaScript unit tests)

The Steady States JavaScript logic has a QUnit test suite in the `xija_steady_states` sibling repository. To run it:

```bash
cd ../xija_steady_states/
conda activate ska3-dev
# Generate fixture data (no real Xija simulation required):
python -m xija_steady_states --fixtures --output-dir test_site
# Serve and open in browser:
python -m http.server 8080 --directory test_site
# Then open http://localhost:8080/tests.html
```

Expected result: **55 tests, 164 assertions, 0 failed**.

### Steady States (Python tests)

```bash
cd ../xija_steady_states/
conda activate ska3-dev
pytest tests/ -v
```

Expected result: **174 passed, 0 failed**.

### Protractor (manual)

The protractor app has no automated test suite. Verify manually after changes:

1. Start the dev server (`python -m http.server 8765` from `meridian/`)
2. Open `http://localhost:8765/protractor/`
3. Confirm the protractor plot renders with colored bands
4. Change the Date and Chips dropdowns — plot and tables should update
5. Change a variable limit dropdown — plot re-renders without a network fetch
6. Expand the "Steady States" sidebar section — model links should appear (requires `steady_states/data/` to exist)
7. Click a model link — navigates to `steady_states/?model=<nav_id>` and pre-selects that model

## Navigation

The sidebar is shared across both apps:

- **Pitch Sensitivity** — links to `protractor/`; collapsed "Steady States" section below it
- **Steady States** — expanded by default; 16 model links load content in-place (SPA, no page reload)

Clicking a model in the Steady States sidebar updates the chart and table without a full page load. Clicking a model link from the protractor sidebar navigates to `steady_states/?model=<nav_id>`, which auto-selects that model on load.

## Data Notes

- `data/` (protractor) and `steady_states/data/` (steady states) contain large binary/JSON files. Do not read individual scenario files into context — use the schema descriptions in `CLAUDE.md` instead. The `manifest.json` in each directory is small and may be read freely.
- Protractor scenario files may contain literal `NaN` (not valid JSON). The app replaces `NaN` with `null` before parsing.
- Steady states data files are standard JSON with no `NaN` values.
